### Касянчук С. ПР-4-2

Цей репозиторій містить практичні роботи та проєкти, виконані в рамках курсу **"Емпіричні методи програмної інженерії"**.

## Структура репозиторію

Проєкт розділений на окремі модулі, кожен з яких демонструє різні аспекти розробки, тестування та моніторингу ПЗ.

### 1. [Task 1: Unit Testing & Verification](./1_task)
Розробка та тестування математичних алгоритмів з використанням різних стратегій верифікації.

- **Мета:** Реалізація класу `Calculator` з базовими арифметичними операціями та покриття їх тестами.
- **Стратегії тестування:**
    1.  **Standard:** Перевірка базової логіки.
    2.  **Exception Handling:** Тести на обробку помилок (наприклад, ділення на нуль).
    3.  **Parameterized:** Запуск тестів з різними наборами вхідних даних.
- **Файли:** `calculator.py`, `test_calculator.py`.

### 2. [Task 2: IoT Monitoring System](./2_task)
Розробка комплексної системи для збору, зберігання та візуалізації метрик роботи емульованих IoT-пристроїв.

- **Мета:** Система імітує роботу розумного будинку (датчики температури та вологості). Дані збираються, зберігаються в NoSQL базі та візуалізуються.
- **Архітектура:**
    - **App (Flask):** Генерація даних та API для авторизації.
    - **Database (MongoDB):** Зберігання часових рядів та показників.
    - **Monitoring (Prometheus):** Збір метрик (Scraping).
    - **Visualization (Grafana):** Побудова графіків та дашбордів.

## Інструкція із запуску

### Завдання 1: Unit Tests
Необхідно мати встановлений **Python** та бібліотеку `pytest`.

```bash
cd 1_task
pip install pytest
pytest -v
```
### Завдання 2: IoT Monitoring
Необхідно мати встановлений Docker Desktop.

```bash
cd 2_task
docker-compose up --build
```
